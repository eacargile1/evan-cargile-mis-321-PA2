Introduction

321 Consulting will design, develop, and deliver a full-stack web-based scheduling and booking system for Legendary Training Services (LTS). The system will replace the client's existing manual scheduling process and provide a centralized platform for trainers, pet owners, and administrators to manage training sessions, bookings, and business reporting.

Purpose and Objectives

The purpose of this project is to automate LTS's scheduling and booking operations, improving efficiency for trainers and the experience for pet owner customers. The project objectives are to:

  • Deliver a fully functional web application accessible to three user roles: Admin, Trainer, and Pet Owner.
  • Allow trainers to create and manage group classes and individual training sessions with pricing, capacity limits, and minimum enrollment thresholds.
  • Allow pet owners to browse, filter, and book available sessions and training packages online.
  • Provide administrators with a reporting suite covering bookings, revenue, trainer payouts, offerings, and customer data.
  • Enforce a 3% platform fee on each booking with automatic trainer payout calculation at 97%.

Tasks

321 Consulting will complete the following tasks as part of this project:

  • Design and implement a REST API backend using ASP.NET Core 9 with Entity Framework Core and SQLite.
  • Build a frontend using HTML, CSS, and JavaScript with role-specific portals for Admin, Trainer, and Pet Owner.
  • Implement JWT-based authentication and BCrypt password hashing for all user accounts.
  • Develop a Browse Sessions page with filtering by session type, breed, and date range.
  • Build a monthly calendar view displaying all available sessions with color-coded indicators.
  • Implement a trainer review and rating system with admin moderation (flag/unflag).
  • Develop all six admin reports: Bookings, Revenue, Business (payouts + repeat customers), Offerings, Pets, and Flagged Reviews.
  • Seed the database with realistic sample data for demonstration and testing purposes.
  • Produce all required project documentation: User Stories, Requirements Traceability Matrix, Test Plan, Test Logs, and this Statement of Work.
  • Execute the test plan, log results, and resolve identified defects prior to handoff.

The following tasks are not included in this project's scope and will not be performed by 321 Consulting:

  • Live payment processing or integration with a payment gateway (e.g., Stripe, PayPal).
  • SMS or email notifications for bookings, cancellations, or reminders.
  • Native iOS or Android mobile applications.
  • Integration with external CRM, accounting, or point-of-sale systems.
  • Automated session cancellation when minimum enrollment is not met.

  • Test Plan outlining testing strategy and sample test cases

Acceptance Criteria

The project will be accepted when all of the following conditions are verified:

  • Pet owners can browse, filter, and book sessions; fully booked sessions cannot be booked.
  • Trainers can create profiles, schedule sessions with a minimum price of $30, create packages, and view reviews.
  • The 3% platform fee and 97% trainer payout are calculated correctly across all reports.
  • Administrators can view all six reports with accurate data and correct filtering behavior.
  • Authentication is enforced; all protected pages redirect unauthorized users to the login page.
  • The session calendar renders correctly with monthly navigation and day-level session detail.
  • Test logs show a minimum of 35 executed test cases with a pass rate of 95% or higher.

Roles and Responsibilities

Legendary Training Services (Client) agrees to:

  • Provide and approve business requirements prior to development.
  • Clarify pricing rules, breed policies, enrollment minimums, and edge cases in a timely manner.
  • Review and approve wireframes and major deliverables within 3 business days of submission.

321 Consulting (Vendor) agrees to:

  • Design and develop the application in accordance with the agreed scope.
  • Produce all documentation deliverables listed above.
  • Execute the test plan and resolve all defects identified prior to handoff.
  • Provide a walkthrough of the completed application to the LTS team.

Assumptions

  • Live payment processing is not required for this engagement; fee calculations are tracked in the database only.
  • The application will be hosted in a development or academic environment for the duration of this project.
  • Authentication is email and password based; OAuth or single sign-on is not required.
  • Browser support targets the latest versions of Chrome, Firefox, Edge, and Safari.
